## Problems

Each problem has **two versions**:

* **Unsolved version**

  * Contains the problem statement
  * May include partial code or TODOs

* **Solved version**

  * Complete implementation
  * Focused on readability
  * Uses simple logic and clear variable names

Problems are intentionally small and incremental.

---

## Commit Strategy

Each problem should be committed separately.

Examples:

```
feat: add problem 1 unsolved version
feat: add problem 1 solved version
fix: correct loop condition in problem 2
```
